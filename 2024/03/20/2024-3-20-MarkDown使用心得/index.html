MarkDown的使用

2024-03-20

菜鸟教程已经介绍得非常详尽了，这里我将列出一些自己常用的格式语法和失误。

插入图片

![alt 属性文本](图片地址 "可选标题")

具体在本博客中的应用就是 

![miku](https://github.com/Refyuan/refyuan.github.io/raw/main/covers/mikuwink.gif "mikuwink")

经测试，jpg，png，gif格式都支持。

插入代码段

代码行：`这里写代码不会正常执行`

要注意是右引号（esc键下面那个）包裹，不是普通引号。 

代码段：四个空格或tab键

有个很生草的事情，就是po第二篇博客的时候每个段落都成了代码块，没有正常渲染。原因是我偷懒直接按了tab键表示段首空格。

隐藏文本

我想要做到像steam评论那样用黑色方块隐藏文本，在鼠标悬停于方块时显示文本。 

<!-- 隐藏文本 -->

失败。隐藏确实隐藏了，但是相当于被注释掉了，无法正常显示。

<p hidden>隐藏文本</p>

失败。这是html隐藏文本的语句，而MarkDown似乎并不支持这个语法。

<details>
<summary>点击查看详细内容</summary>
展开的内容
</details>

点击查看详细内容 展开的内容

成功。如果我把“点击查看详细内容”换成空字符，就可以达成。

展开的内容

但和我想展现的效果并不完全一致。

在这里我们知道了MarkDown本身比较轻量，并不支持这种语法。
只能另寻他路了。

使用插件hexo-spoiler。

站点根目录配置填写：npm install hexo-spoiler --save

_config.yml文件中添加

spoiler:
  style: blur # 或者box
  color: black # 仅当 style 为 box 时起效
  p: false # 没懂啥意思，不管它

标准格式：option:value text…

默认配置效果 → 默认配置效果

糊里糊涂 → 糊里糊涂

黑不溜秋 → 黑不溜秋

红光闪闪 → 红光闪闪
